# Feature — Luo feature-spesifikaatio

Luo kattava feature-spec annetusta ideasta: $ARGUMENTS

## Vaiheet

### 1. Ymmärrä nykytila
- Lue `index.html` ja tunnista mihin kohtaan uusi ominaisuus liittyy
- Listaa olemassa olevat funktiot ja taulut joita feature koskee
- Tunnista mahdolliset ristiriidat tai päällekkäisyydet

### 2. Käyttäjätarina
Kirjoita 1–3 käyttäjätarinaa muodossa:
> **Käyttäjänä** haluan [toiminto], **jotta** [hyöty].

### 3. Hyväksymiskriteerit
Listaa konkreettiset kriteerit jotka määrittävät milloin feature on valmis:
- [ ] Kriteeri 1
- [ ] Kriteeri 2
- ...

### 4. Tekniset reunaehdot
- **Taulut**: tarvitaanko uusia Supabase-tauluja tai sarakkeita?
- **Funktiot**: mitkä olemassa olevat funktiot pitää muokata, mitä uusia tarvitaan?
- **UI**: mitä uusia elementtejä tarvitaan, mihin välilehteen?
- **Käännökset**: mitä uusia t()-avaimia tarvitaan (fi + en)?
- **RLS**: tarvitaanko uusia Row Level Security -sääntöjä?

### 5. Rajaukset
- Mitä feature EI sisällä (scope out)
- Tunnetut riskit tai epävarmuudet

### 6. Riippuvuudet
- Edellyttääkö toisen featuren valmistumista
- Vaikuttaako muihin olemassa oleviin ominaisuuksiin

## Raportti
Kokoa kaikki yllä oleva selkeäksi speksiksi. Käytä suomea.
